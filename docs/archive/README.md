# Archive Directory

This directory contains archived documentation that is no longer actively maintained but kept for historical reference.

## Directory Structure

```
archive/
├── README.md                    # This file
├── debug/                       # Debug and diagnostic reports
├── status/                      # Status reports and fix summaries
├── analysis/                    # Analysis and investigation reports
├── audits/                      # Audit reports
└── guides/                      # Historical guides and templates
```

## Categories

### Debug (`debug/`)
Debug reports, diagnostic outputs, and forensic analysis documents from troubleshooting sessions.

**Examples:**
- Debug output files
- Diagnostic reports
- Forensic analysis documents
- Code search results

### Status (`status/`)
Status reports, fix summaries, migration summaries, and completion reports.

**Examples:**
- System status reports
- Fix summaries
- Migration completion reports
- Implementation summaries

### Analysis (`analysis/`)
Analysis reports, investigation summaries, and transition analysis documents.

**Examples:**
- Transition flow analysis
- Layout analysis
- Component analysis
- Investigation summaries

### Audits (`audits/`)
Audit reports and verification documents.

**Examples:**
- Code audits
- System audits
- Verification reports

### Guides (`guides/`)
Historical guides, templates, and reference documents.

**Examples:**
- Implementation guides
- Prompt templates
- Historical reference documents

## Purpose

Files in this archive are:
- **Historical records** of completed work
- **Reference material** for understanding past decisions
- **Not actively maintained** - may contain outdated information
- **Kept for context** - useful for understanding project evolution

## Maintenance

- Files are moved here when they are no longer actively used
- Do not delete archived files without careful consideration
- Archive structure may be reorganized as needed
